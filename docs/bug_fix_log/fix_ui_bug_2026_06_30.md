# UI 与 Git 追踪 Bug 修复报告 (2026-06-30)

本报告详细记录了在 2026-06-30 修复的系统 UI 视觉显示缺陷以及 Git 追踪配置项问题。

---

## 修复缺陷列表

### 1. 界面按钮与标签的 Emoji 图标丢失

* **缺陷描述**：
  主界面的 Tab 标签栏（如回测引擎、数据获取等）以及设置、工具菜单和数据管理按钮的 Emoji 图标全部消失，仅显示未翻译的原始键名（如 `tabs.backtest_engine`）。
* **原因分析**：
  前序修改中将硬编码文本替换成了本地化翻译键，但项目 `.gitignore` 误将 `config/` 配置文件夹整体忽略，导致翻译语言文件（`zh_CN.json` 和 `en_US.json`）未能提交至 Git。此外，原代码的 `tr()` 逻辑按点号分割键以查找嵌套的 JSON 结构，而旧的翻译文件是扁平字典，导致即使重新补全文件也无法正确解析。
* **解决方案**：
  1. 修改了 [.gitignore](file:///d:/Personal/quant/QuantDataBridge-main/.gitignore#L49-L50) 以将语言包路径 `!config/languages/` 从忽略列表中排除。
  2. 新增并提交了重构为嵌套结构的本地化翻译包 [zh_CN.json](file:///d:/Personal/quant/QuantDataBridge-main/config/languages/zh_CN.json) 与 [en_US.json](file:///d:/Personal/quant/QuantDataBridge-main/config/languages/en_US.json)。
* **效果**：
  主界面、菜单栏、数据管理按钮及所有选项卡的 Emoji 图标和翻译内容全部恢复正常。

### 2. 回测页面“刷新列表 (Refresh List)”按钮隐形/消失

* **缺陷描述**：
  回测引擎左侧参数面板中，选择信号文件的 combo box 下方对应的刷新按钮不可见（呈空白条状）。
* **原因分析**：
  1. **暗黑模式背景冲突**：该按钮未显式设置样式表，在 Windows 系统的暗黑风格下继承了原生黑色前景色，导致在深色背景下文本无法显现。
  2. **超长文件名导致布局被裁剪**：`QComboBox` 默认以其包含的最长选项文本（长文件路径）作为其 size hint 宽度（达 700+ 像素），将整个左侧参数面板撑宽并超出了 viewport 可视区（320 像素）。由于禁用了水平滚动条，居中对齐的按钮文字被渲染在可视区右侧，无法被用户看到。
* **解决方案**：
  1. 限制 `self.file_combo` 控件的 `maximumWidth` 为 `200` 像素，防止布局被无限撑宽。
  2. 将按钮的文本使用本地化键 `tr("alignment.buttons.refresh")`（解析为带图标的 `⟳ 刷新列表`）。
  3. 为 `refresh_btn` 配置显式样式（`color: white; background-color: #37474F`）并清除可能导致裁剪的 padding 属性，设置最低高度为 `30` 像素。
* **效果**：
  按钮的图标和文字重新在左侧面板居中显现，高度与样式风格完全贴合 `qdarktheme` 规范。

### 3. Datacenter 运行时数据文件被 Git 追踪

* **缺陷描述**：
  `datacenter/` 目录下由系统运行时下载和对齐生成的本地 `.parquet` 文件频繁被 Git 提示为已修改（`modified`）或未追踪（`untracked`）。
* **原因分析**：
  `datacenter/` 目录存放的是运行时动态数据，但最初没有在 `.gitignore` 中对其进行配置。同时有数个历史 Parquet 文件已被加入到 Git 索引缓存中，导致其即使在本地发生了改动也会不断被 Git 追踪。
* **解决方案**：
  1. 在 [.gitignore](file:///d:/Personal/quant/QuantDataBridge-main/.gitignore#L45-L48) 中增加了对整个 `datacenter/` 目录的忽略配置。
  2. 运行了 `git rm -r --cached datacenter` 清除了 Git 的索引跟踪，且安全保留了本地已生成的历史文件。
* **效果**：
  Git 状态恢复清爽，后续系统下载、修改或生成的任何本地数据文件都将处于安全的忽略状态，不会被误提交至 GitHub 仓库。
